<objective>
Build the hard filter layer for the matching engine: a pure-Python module that
filters a list of job dicts by job type, sponsorship requirement, and location
preference before scoring runs.

sponsorship requirements, and locations without touching the ANN retrieval query.

Create src/wekruit_matching/matching/filters.py with the following:

1. LOCATION_ALIASES dict[str, str] — maps raw strings (lowercased) to canonical bucket:
   {
     "sf": "san francisco",
     "san francisco": "san francisco",
     "san francisco, ca": "san francisco",
     "sf, ca": "san francisco",
     "nyc": "new york",
     "new york": "new york",
     "new york, ny": "new york",
     "ny": "new york",
     "la": "los angeles",
     "los angeles": "los angeles",
     "los angeles, ca": "los angeles",
     "seattle": "seattle",
     "seattle, wa": "seattle",
     "austin": "austin",
     "austin, tx": "austin",
     "boston": "boston",
     "boston, ma": "boston",
     "chicago": "chicago",
     "chicago, il": "chicago",
     "remote": "remote",
   }

2. normalize_location(loc: str) -> str
   Lowercase the input, strip whitespace, look up in LOCATION_ALIASES, return
   canonical bucket if found, otherwise return the lowercased-and-stripped input.

3. _job_location_buckets(location_raw: str) -> set[str]
   Split location_raw on ";" and ",", normalize each token, return set of buckets.
   Always include "remote" check: if any token normalizes to "remote", the set
   contains only {"remote"} (remote is universal — no other filtering needed).

4. filter_by_job_type(jobs: list[dict], job_type: JobType) -> list[dict]
   If job_type is ANY, return jobs unchanged.
   Map: INTERN -> source_repo == "Summer2026-Internships"
        NEW_GRAD -> source_repo == "New-Grad-Positions"
   Return only rows matching the mapped source_repo value.

5. filter_by_sponsorship(jobs: list[dict], requires_sponsorship: bool) -> list[dict]
   If requires_sponsorship is False, return jobs unchanged.
   Otherwise keep only rows where jobs["sponsorship"] is exactly True (bool).
   Rows where sponsorship is False OR None are excluded.

6. _preferred_buckets(preferred_locations: list[str]) -> set[str]
   Normalize each entry in preferred_locations using normalize_location.
   Return as a set.

7. filter_by_location(jobs: list[dict], preferred_locations: list[str]) -> list[dict]
   If preferred_locations is empty, return jobs unchanged.
   Compute preferred_buckets = _preferred_buckets(preferred_locations).
   If "remote" in preferred_buckets, return all jobs (user is open to remote anywhere).
   For each job: compute job_buckets = _job_location_buckets(job["location_raw"] or "").
   If "remote" in job_buckets, include the job (remote jobs match any preference).
   If job_buckets & preferred_buckets is non-empty, include the job.
   Otherwise exclude.

8. apply_hard_filters(jobs: list[dict], profile: UserProfile) -> list[dict]
   Chain: jobs -> filter_by_job_type -> filter_by_sponsorship -> filter_by_location.
   Each function receives the output of the previous.
   Import UserProfile from wekruit_matching.models.user_profile.
   Return the final filtered list.

Use loguru for one info-level log at the end of apply_hard_filters:
  logger.debug("Hard filters: {} -> {} jobs", len(original), len(result))

No DB imports. No psycopg. No SQLAlchemy. Pure Python.
