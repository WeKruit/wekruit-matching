```
#!/usr/bin/env python
"""End-to-end pipeline exerciser for WeKruit Matching Engine.

Runs: scrape -> enrich -> embed -> match -> feedback
Requires a live Postgres instance and valid .env credentials.

Usage:
    uv run python scripts/e2e_test.py
"""
import sys
from loguru import logger
from wekruit_matching import get_matches, record_feedback
from wekruit_matching.scraper.run import scrape_all
from wekruit_matching.enrichment.run import enrich_all
from wekruit_matching.embedding.run import embed_all
from wekruit_matching.models.user_profile import UserProfile, JobType, CompanySizePreference
```

Pipeline steps (call each in sequence, log stats after each):

1. SCRAPE: Call scrape_all(), log per-repo inserted/updated/unchanged/stale counts. If total inserted+updated == 0, log a warning "No new or updated jobs — database may already be current" but continue.

2. ENRICH: Call enrich_all(), log enriched/failed counts.

3. EMBED: Call embed_all(), log embedded/failed counts.

4. MATCH: Build a test profile and call get_matches():
   - user_id="e2e-test-user"
   - skills=["Python", "machine learning", "SQL"]
   - preferred_job_type=JobType.INTERN
   - preferred_locations=["Remote", "SF", "NYC"]
   - requires_sponsorship=False
   - preferred_company_size=CompanySizePreference.ANY
   - top_n=10
   Print a header "=== TOP MATCHES ===" then for each result print:
     f"  [{i+1}] {job['role_title']} @ {job['company_name']} | score={job['score']:.3f} | {job['location_raw']}"
   Print the signals dict for the first result only (detailed breakdown).
   If matches is empty, log a warning "No matches returned — check that jobs are enriched and embedded".

5. FEEDBACK: If matches is non-empty, call record_feedback() on the first match:
   - user_id="e2e-test-user"
   - job_id=matches[0]["job_id"]
   - reaction="like"
   Log "Recorded 'like' feedback for job_id={job_id}"

6. Print summary:
   - scrape stats, enrich stats, embed stats, match count, feedback recorded (bool)
   - Final line: "E2E complete. All pipeline steps ran successfully."

Error handling: Wrap the entire pipeline in try/except Exception. On failure, log the exception and sys.exit(1). On success, sys.exit(0).

All logging uses loguru: logger.remove() at top, logger.add(sys.stderr, level="INFO") to reset to clean output.

Do NOT seed or insert the test user profile into the DB from within this script — record_feedback handles non-existent users gracefully (logs a warning and returns). If the caller needs a real user profile row, that is an operational concern outside the E2E scope.
