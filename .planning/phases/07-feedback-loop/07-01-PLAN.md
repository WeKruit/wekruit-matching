    ```
    """Feedback handler: record user reactions and update profile state.

    record_feedback(user_id, job_id, reaction, conn) — public API
    """
    ```

    Function signature:
    ```python
    def record_feedback(
        user_id: str,
        job_id: str,
        reaction: str | ReactionType,
        conn: psycopg.Connection | None = None,
    ) -> None:
    ```

    Internal steps (executed inside a single connection — use conn injection pattern
    matching get_connection() in matcher.py: if conn is not None use it, else get_connection()):

    **Step 1 — Insert feedback row:**
    ```sql
    INSERT INTO feedback (user_id, job_id, reaction, recorded_at)
    VALUES (%s, %s, %s, NOW())
    ON CONFLICT DO NOTHING
    ```
    Use ON CONFLICT DO NOTHING so duplicate reaction recordings are idempotent
    (ix_feedback_user_job index covers user_id + job_id; same reaction can be
    re-recorded safely without raising).

    **Step 2 — Fetch job's company_name and embedding:**
    ```sql
    SELECT company_name, embedding FROM jobs WHERE job_id = %s
    ```
    Store as job_row. If job_row is None (job not found), log a warning and return early.

    **Step 3 — Branch on reaction:**

    If reaction == "like" or ReactionType.LIKE:
      a. Append company_name to liked_companies (via Postgres array_append):
         ```sql
         UPDATE user_profiles
         SET liked_companies = array_append(liked_companies, %s),
             updated_at = NOW()
         WHERE user_id = %s
         ```

      b. If job_row["embedding"] is not None:
         - Fetch current affinity_embedding from user_profiles:
           ```sql
           SELECT affinity_embedding FROM user_profiles WHERE user_id = %s
           ```
         - If current affinity is None (first like):
             new_affinity = list(job_row["embedding"])
             # No normalization needed — job embeddings are already unit-norm from OpenAI
         - Else (subsequent like):
             existing = np.array(current_affinity, dtype=np.float64)
             new_signal = np.array(job_row["embedding"], dtype=np.float64)
             blended = 0.7 * existing + 0.3 * new_signal
             norm = np.linalg.norm(blended)
             new_affinity = (blended / (norm + 1e-9)).tolist()
         - Update affinity_embedding:
           ```sql
           UPDATE user_profiles
           SET affinity_embedding = %s::vector,
               updated_at = NOW()
           WHERE user_id = %s
           ```

    If reaction == "dislike" or ReactionType.DISLIKE:
      Append company_name to disliked_companies:
      ```sql
      UPDATE user_profiles
      SET disliked_companies = array_append(disliked_companies, %s),
          updated_at = NOW()
      WHERE user_id = %s
      ```

    If reaction == "applied":
      No profile update — feedback row already inserted in Step 1.

    **No commit inside record_feedback** — commit is the caller's responsibility
    (matches the pattern in enrichment/worker.py and embedding/worker.py).

    **Imports needed in handler.py:**
    - psycopg (for type hint)
    - numpy as np
    - loguru.logger
    - wekruit_matching.db.connection.get_connection
    - wekruit_matching.models.feedback.ReactionType
    - pgvector.psycopg.register_vector (call register_vector(conn) before any vector column write/read)

    **register_vector note:** Call register_vector(conn) once at the top of the
    internal _run helper (same pattern as matcher.py _fetch_ann_candidates) so
    pgvector type codec is active for both the embedding SELECT and UPDATE.
