DB query for ANN retrieval:
```sql
SELECT
    job_id, source_repo, company_name, role_title, primary_url,
    location_raw, date_posted_raw, status, first_seen_at, last_seen_at,
    industry, company_size, required_skills, sponsorship,
    embedding::text,   -- cast to text then parse back to list[float]
    embedding_model
FROM jobs
WHERE status = 'active'
  AND embedding IS NOT NULL
ORDER BY embedding <=> %s::vector
LIMIT %s;
```
-- Pass the query_embedding as a Python list; psycopg3 will serialize it.
-- Use register_vector(conn) from pgvector.psycopg before executing.
-- The ANN fetch limit is top_n * 4 (e.g., 30 * 4 = 120 candidates).
-- Hard filters and scoring run in Python after the DB fetch.

Query embedding composition:
- If profile.skills is non-empty: query = ", ".join(profile.skills)
- If profile.skills is empty: query = "software engineer"   # generic fallback
- Then call embed_text(query)

Affinity embedding (cold-start handling):
- If profile.affinity_embedding is not None, use it as query_embedding instead
  of calling embed_text. This is the Phase 7 hook — Phase 6 just passes through.
- If profile.affinity_embedding is None: embed profile.skills as described above.
- Cold-start (no affinity_embedding, no skills) still works — feedback_boost
  returns 0.5 for all jobs; the other 6 signals drive ranking.
