# Milestone v1.1: Internal UI Foundation

**Status:** SHIPPED 2026-03-31  
**Phases:** 9-11  
**Total Plans:** 3

## Overview

This milestone turned the internal matching pages into a coherent WeKruit jobs console. The work focused on shared shell structure, tokenized styling, accessibility, responsive jobs browsing, and customer-facing-ready hierarchy across Jobs, Stale, Stats, and Pipeline.

## Phases

### Phase 9: Console Shell & Design Tokens

**Goal**: All internal pages share one WeKruit-aligned shell with clear titles, navigation, accessible structure, and reusable visual primitives instead of scattered hard-coded styles  
**Depends on**: Phase 8  
**Plans**: 1

Plans:

- [x] 09-01: Shared shell, token layer, semantic structure, and render-level verification

**Details:**
- Shared page shell across Jobs, Stale, Stats, and Pipeline
- Clear current-page title, summary region, and consistent navigation structure
- Visible focus, form labels, and valid heading hierarchy
- Internal/external mode support via shared shell/token layer

### Phase 10: Jobs Browsing UX Overhaul

**Goal**: Users can browse active and stale jobs from a clear, responsive interface that preserves operational density without collapsing into a desktop-only table  
**Depends on**: Phase 9  
**Plans**: 1

Plans:

- [x] 10-01: Jobs/stale responsive browsing, status language, and pagination semantics

**Details:**
- One coherent filter region for status, source, industry, and search
- Mobile card layout for narrow viewports
- Text-based freshness and processing state communication
- Touch-friendly pagination that preserves filter context

### Phase 11: Customer-Facing Readiness & Final Polish

**Goal**: Stats and Pipeline read as calm, product-quality informational pages and the whole console is structurally ready for later customer-facing mode work  
**Depends on**: Phase 10  
**Plans**: 1

Plans:

- [x] 11-01: Stats/pipeline hierarchy, dual-surface readiness, and cross-page consistency polish

**Details:**
- Stats page communicates headline inventory metrics first
- Pipeline communicates pending work and recent activity in product language
- Shared layout, spacing, and section rules across all pages
- Explicit structural support for internal and external surface modes

## Milestone Summary

**Key Decisions:**

- Keep the UI server-rendered inside `internal_ui.py` rather than introducing a new frontend stack
- Model internal vs external as surface modes on one shell rather than separate page trees
- Prioritize shell, semantics, and responsive structure before visual polish

**Issues Resolved:**

- Removed page-by-page styling drift
- Fixed missing page headings and labels
- Removed status shorthand like `Y/--` in favor of explicit language
- Eliminated horizontal-scroll-only browsing as the mobile fallback for jobs

**Issues Deferred:**

- Browser-based visual QA snapshotting
- Richer charting and job detail surfaces

**Technical Debt Incurred:**

- Nested repo GSD root detection still needs an upstream fix; milestone execution used local planning context directly
